# Fix crash on hosts with null values in meta_data

---------- | ---
date       | 2026-07-21T12:36:03.230004+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | rest-api
level      | 1
compatible | yes

The REST API crashed when serializing the attributes of a host or
folder whose <tt>meta_data</tt> attribute contained explicit null
values for <tt>created_at</tt>, <tt>updated_at</tt> or
<tt>created_by</tt> instead of omitting the keys entirely.

This also affected the Dynamic host configuration (DCD), which uses
these endpoints and crashed when processing such hosts.

The REST API now treats null values in <tt>meta_data</tt> the same as
missing keys and omits the corresponding fields from the response.
