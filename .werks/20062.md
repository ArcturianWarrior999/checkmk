# Views: graph legend header overlapped first row when grouping is used

key        | value
---------- | ---
date       | 2026-07-17T11:21:32+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | multisite
level      | 1
compatible | yes

When a table view contained the "Service graphs with time range previews"
column and was grouped by a column (for example by host name), the legend
below each graph was rendered incorrectly: the header row holding the
"Minimum", "Maximum", "Average" and "Last" values was shifted downwards and
covered the first entry of the legend, obscuring the graph values.

The legend now renders correctly again, whether or not the view uses grouping.
