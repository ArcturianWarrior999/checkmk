# Custom snap-ins in the default sidebar no longer crash the user interface

key        | value
---------- | ---
date       | 2026-06-08T05:20:54+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | multisite
level      | 1
compatible | yes

Administrators can predefine the snap-ins shown in the sidebar for new users.
If this default configuration contained a custom snap-in (one created under
"Customize > Custom snap-ins"), the whole user interface crashed when a user
without a personal sidebar configuration logged in.

This has been fixed. Custom snap-ins can now be used in the default sidebar
configuration again. Entries that refer to a snap-in which does not exist (for
example a typo or a removed snap-in) are silently ignored instead of breaking
the page.
