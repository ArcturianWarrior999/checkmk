# Custom host notes URL missing in service email notifications

After upgrading to 2.4, HTML email notifications for service problems no longer
contained the "Custom host notes URL". Only the "Custom service notes URL" was
shown.

Before 2.4, a service notification listed both the custom host notes URL and the
custom service notes URL when the "Custom host/service notes URL" element was
selected in the notification's HTML email parameters.

This restores the previous behaviour: service notifications now show both URLs
again, while host notifications continue to show the custom host notes URL.
