# Don't show LDAP connection tests on validation errors

key        | value
---------- | ---
date       | 2026-07-14T12:19:24.318354+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

When a LDAP connection was edited and then saved with the "Save & test" button,
but the input validation failed (e.g. a char in the port number) the connection
was not saved but the tests still ran with the old connection config.
A red error was still shown explaining that some validation failed but the tests
might still show a green state.
This could cause some confusion.

With this Werk no tests are run if the input validation fails.
